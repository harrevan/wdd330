MDN Using Fetch

Fetch API

Prrovides a JavaScript interface for accessing and manipulating parts of HTTP pipeline (requests, responses, etc.)

Provides global fetch() method providing a way to fetch resources asynchronously across the network.

Provides a better alternative to XMLHttpRequest.

fetch differs from jQuery.ajax() in 3 main ways:
1.) Promise returned from fetch() won't reject HTTP on error status, only on network failure or if request is prevented from completing.
2.) fetch() won't receive cross-site cookies. 3.) fetch w+on't send cookies.

Supplying request options

fetch() can accept a second paramater, an init object that allows you to control certain settings

Sending request with credentials included

add credentials: 'include' to init object passed to the fetch method.

Uploading JSON data

fetch() can POST JSON-encoded data.

Files

fetch() can be used to upload one or multiple files when used with correct input elements

Headers

Headers interface allows you to create a headers object. A headers object is a multi-map of names to values.

Response Objects

Response instances are returned when fetch() promises are resolved. Most common response proberties are Response.status, Response.statusText, Response.ok

Body

Requests and responses can contain body data. Body is an instance of: ArrayBuffer, ArrayBufferView, Blob, string, URLSearchParams, or FormData.
