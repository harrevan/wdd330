MDN: Validating Forms

What is form validation?

Browser and/or web server checks submited data to make sure it is formatted correctly and properly constrained. Typically involves providing helpful error messages to users.

Validation in the browser is client side, on the server is called server-side.

Reasons to validate forms: to get the right data the right way, protect user data, protect ourselves.

Types of client-side validation

Built-in form validation: uses HTML5 form validation features. Doesn't involve javascript usually. better performance, less customizable.

JavaScript validation: coded validation using JavaScript. Completely customizable, but has to be created, or library must be used.

Using built in form validation

Use validation attributes on form elements like required, minlength, maxlength, min, max, type, pattern

:valid and :invalid CSS pseudo-class allows style to be applied to valid or invalid form elements
