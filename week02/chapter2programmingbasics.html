Grammar
Note, statements in javascript are ended with either a new line or a semi-colon. Best practice dictates
the use of semi-colon with the new line.

Data Types
6 primitive:
  String
  Symbol
  Number
  Boolean
  Undefined
  null
Objects are the 7th type: arrays, functions, object literals.

typeof operator shows value of value. EX: typeof 'hello';

Variables
const - declares variable not to be reassigned
let - variable might be reassigned

If a non-primitive variable is declared with const, it is not immutable. (Underlying data in object)
can change, aka mutated.
EX:
const name = { value: 'Alexa' }; // an object
name.value = 'Siri'; // change the value

const DOES prevent assigning a variable to another object.
EX:
// declare object
const name = { value: 'Alexa' };
// attempt to reassign to another object
name = { value: 'Siri' }
produces a TypeError: Assignment to constant variable.
let and const both have block scope

Best to use Camel Case when naming constants, variables, and functions.

Assiging primitive values to a variable changes the value.
Assigning non-primitive values to a variable is done by reference. changes will affect all references to the object.
EX:
const c = { value: 1 };
let d = c; // c.value = 1, d.value = 1
d.value = 2; // c.value = 2, d.value = 2

Use \ as escape characters

Wrapper Objects are created for primitive data types to allow them to have properties and methods.
Properties can be accessed with . or []. For example, name.length or name['length'].
Properties of primitive data types are immutible.

Template Literals
Deliminated using backtick `. EX: `Hello!`
Can use both types of quote marks. Also allows for interpolation. Javascript expressions can be inserted in the string
EX:
const name = `Siri`;
`Hello ${ name }!`;
output: 'Hello Siri!'
const age = 39;
`I will be ${ age + 1 } next year`;
output: 'I will be 40 next year'

Symbols Used to create unique values.
Don't have a literal form. Have to use =Symbol('include a description here');
