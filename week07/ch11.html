Javascript: Novice to Ninja Chapter 11: Further Functions

Function Properties and Methods

Functions can have properties and methods since they are first-class objects.

Call and Apply Methods

call() method used to set value of this inside a function to an object provided as first argument. (See console for examples.)

If the function that is called requires parameters, these shlud be provided to the call() function after the first argument. If the function doesn't refer to an object as this in its body, it can still be called using call(), but null must be the first argument.

apply() works in a similar way, but the arguments of the function are provided as an array.

Custom Properties

You can add your own properties to functions just like any other object.

Custom properties can be used for memoization, or caching

Immediately Invoked Function Expressions

An IIFE is "an anonymous function that, as the name suggests, is invoked as soon as it’s defined"

This can be done by invoking it (parenthesis at end of function) and making it into an expression by surrounding it in parenthesis

Functions that Define and Rewrite Themselves

By assigning an anonymous function to a variable with the same name as the function, the funciton can define, redefine, and call itself.

This is the Lazy Definition pattern, helpful when initialization code is required the first time a function is invoked, and subsequent invocations require a new definition.

Init-Time Branching

Technique used to create functions that rewrite themselves.

Enables functions to work more effectively in the browser.

Recursive Functions

Function that invokes itself until a certain condition is met. Useful for iterative processes.

Callbacks and Promises: see console and examples.js for examples

Async Functions

Functions are preceeded by async, and allow asynchronous code to be written synchronously.

await operator used before function wraps return value in a promise that can be assigned to a variable. The next line of code is not executed until the promise resolves.

Generalized Functions

See examples.js.

Closures

References to variables created inside the scope of another function, but is kept alive and used in another part of the program

"One of the key principles in creating closures is that an “inner” function, which is declared inside another function, has full access to all of the variables declared inside the scope of the function in which it’s declared (the “outer” function)."

Returning Funcitons

"A closure is formed when the inner function is returned by the outer function, maintaining access to any variables declared inside the enclosing function."

Functional Programming

JavaScript is a multi-paradigm language (supports procedural, functional, and object oriented programing paradigms)

Pure Functions

Functional programming uses pure functions. Rules include:

  • Return value should only depend on the values provided as arguments
  • There are no side-effects, doesn't change values or data anywhere else.
  • Referential transparency, given the same arguments a pure function always returns same value.
